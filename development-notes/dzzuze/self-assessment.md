## Описание моей работы

В этом проекте я пытался быть тимлидом, но как мне кажеться, так себе получилось.
В основном отвечал за дизайн, анимации, цвета, фон и т. д. Но котиков добавила Аня.
Ещё могу выделить добавление CI/CD.

### Основные зоны ответственности:

- CI/CD
- Login Page
- Sign Up Page (RegisterPage)
- Дизайн приложения.

## Технологии и инструменты

- React + React Router DOM - для маршрутизации и условного рендеринга страниц.

- Tailwind CSS - для стилизации, включая кастомную цветовую палитру, тёмную тему, адаптивные правки.

- Vitest + React Testing Library (RTL) - для написания модульных тестов (LoginPage, Header).

- GitHub Actions - настройка CI/CD и деплоя.

- Jira / GitHub Issues - управление задачами, трекинг прогресса, распределение обязанностей.

## Процесс и ключевые этапы

Организация работы
Вёл дневник, создавал Issues для себя и коллег.

Настройка проекта и дизайн-системы
Добавил Tailwind, создал глобальную конфигурацию стилей. Определил цветовую палитру (тёмная тема с серо-жёлтыми элементами), добавил фоновых «котиков».

Разработка страниц аутентификации
Создал дизайн LoginPage, RegisterPage, позже стилизовал ResetPasswordPage. Работал с формами, валидацией, обработкой ошибок. Восстанавливал отображение этих страниц после конфликтных PR (слетали стили и переводы). Добавил анимацию загрузки и проверки авторизации.

Создание общей UI-инфраструктуры
Разработал переиспользуемый компонент кнопки. Настроил хедер и футер: динамическое меню для авторизованных/неавторизованных пользователей, user nav menu с иконками компаса и карты, анимацию при наведении. Удалил дублирующийся «бургер» при низком разрешении, убрал горизонтальный скролл, который вызывал сдвиг страницы.

Работа с тестами
Написал 5 тестов для LoginPage (успешный вход, обработка ошибок, отображение ключевых элементов, изменение полей ввода, проверка ссылки «Meow!»). Затем написал новые тесты для Header. Исправлял сообщения ошибок в тестах страницы восстановления пароля. Разбирался с нюансами RTL и асинхронными проверками.

Code review и разрешение конфликтов
Регулярно проверял PR коллег (например, PR по Header от Анны), давал комментарии по стилизации и функциональности. Решал конфликты слияния, которые ломали страницы и переводы.

"О нас"
Оформил страницу About (информация о разработчиках).

## Что было сложным:

- Конфликты в PR - после крупных изменений слетали стили, логика отображения страниц и переводы. Приходилось вручную восстанавливать работоспособность.

- Тестирование - требовалось время, чтобы разобраться с RTL и моками, особенно для асинхронных операций (авторизация). Некоторые тесты неожиданно проходили, хотя должны были падать - пришлось переписывать.

- Роутинг и условный рендеринг – при удалении/возврате кнопок навигации нужно было следить, чтобы не сломать переходы между страницами и меню пользователя.

- Адаптивные «костыли» - устранение горизонтального скролла и лишнего бургера потребовало отладки Tailwind и медиа-запросов.
