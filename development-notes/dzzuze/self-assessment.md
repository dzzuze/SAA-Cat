## Описание моей работы

В этом проекте я был тимлидом и преимущественно дизайнером.
Отвечал за CI/CD и визуальную часть.

### Основные зоны ответственности:

- CI/CD
- Дизайн приложения.

## Технологии и инструменты

- React - для рендеринга страниц.

- Tailwind - для стилизации.

- Vitest + React Testing Library (RTL) - для написания тестов.

- GitHub Actions - настройка CI/CD и деплоя.

- GitHub Issues - управление задачами.

## Процесс и ключевые этапы

### Организация работы
- Вёл "дневник", создавал Issues для себя и коллег.

### Настройка дизайн-системы
- Добавил Tailwind и определил цветовую палитру.

### UI-дизайн и верстка интерфейса
- Реализовал визуальную стилистику и UI-компоненты для основных страниц сервиса: Header, Footer, AboutPage, LoginPage, RegisterPage, PasswordRecoveryPage, UserInformationPage.

### Работа с тестами
- Написал 5 тестов для LoginPage (успешный вход, обработка ошибок, отображение ключевых элементов, изменение полей ввода, проверка ссылки «Meow!»).

### Code review и разрешение конфликтов
- Регулярно проверял PR коллег, давал фидбек по стилизации и функциональности. Решал конфликты слияния, влияющие на отображение страниц.

## Что было сложным

- Конфликты в PR - после крупных изменений слетали стили, логика отображения страниц. Приходилось вручную восстанавливать работоспособность.

- Тестирование - требовалось время, чтобы разобраться с тестами, моками, особенно для асинхронных операций (авторизация). Некоторые тесты неожиданно проходили, хотя должны были падать - пришлось переписывать.

- Роутинг.

- "Костыли" - устранение горизонтального скролла и лишнего бургера потребовало отладки Tailwind и "медиа".
