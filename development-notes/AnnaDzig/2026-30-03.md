# Date: 2026-03-30

- **What was done:**

Today I was working on improving the User Page, mainly the part with showing completed lessons.

At first I just fetched data from Firebase and rendered it as a list, but honestly it looked really bad. There were too many small cards and everything felt messy and not like a real product. So I stopped and decided to rethink the UI a bit.

I changed the approach and grouped lessons by topic, like TypeScript, Core JavaScript etc. After that it immediately started to look much better and more structured. It became easier to read and actually looks like something usable.

Then I spent quite some time fixing the layout, because the proportions between the profile form and the progress block were off. On desktop it didn’t feel balanced, and on mobile it also needed better stacking. I adjusted widths and spacing so now both blocks look like part of the same page, not random elements.

Later I refactored everything and moved the progress logic into a separate component `CompletedLessonsCard`. This made the UserPage much cleaner and easier to read. Also added small formatting helpers because raw strings from Firebase looked ugly.

- **Decisions / Notes:**

I decided to keep progress as a separate component, because it will be easier to reuse later and not overload UserPage.

Also decided to keep current Firebase structure for now (array of strings), even though it’s not perfect, because changing backend would take too much time at this stage.

Used real-time updates with onSnapshot so the UI updates automatically.

- **Problems / Questions:**

Main problem was UI. First version looked really bad, like just data dump, not a real interface.

Also hit React lint error about setState inside useEffect, had to change approach with loading state.

Another thing was figuring out how to structure lessons so it doesn’t look chaotic.

- **Plans / Next steps:**

Next I want to show progress inside lessons list and maybe add some kind of progress indicator.

Also later we should probably change Firebase structure to store objects instead of strings.
