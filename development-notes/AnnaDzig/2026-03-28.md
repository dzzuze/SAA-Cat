# Date: 2026-03-28

- **What was done:**

Today I continued building the learning flow and focused on making it fully functional.

I created a separate **LessonsPage**, which opens after clicking Practice and displays all lessons for a topic in the correct order. Each lesson can now be opened individually.

Then I worked on the **LessonPage**, where the lesson content is rendered using MarkdownRenderer. I added a **Complete lesson** button and implemented saving progress to Firebase inside the user's `completed` array.

After that, I added navigation between lessons (**Back** and **Go on** buttons). The navigation is based on lesson order, and the "Go on" button is disabled until the lesson is completed.

Later I fixed an important issue: when a user reopens a lesson, the page did not remember that it was already completed. I updated the logic to fetch the user's progress from Firebase and restore the completion state on page load.

- **Decisions / Notes:**

Decided to:

- keep completion state fully in Firebase (user document)
- use a stable key format: `topicId:lessonId`
- support old data temporarily (title-based keys)
- control navigation based on completion state
- avoid unsafe patterns like `topicId!` and instead handle undefined properly

This makes the flow predictable and aligned with real learning platforms.

- **Problems / Questions:**

Main issues today:

- TypeScript errors caused by incorrect helper usage (wrong arguments)
- mismatch between old and new completion key formats
- initial confusion about where to store and read user progress

Also had to carefully adjust logic without breaking existing dashboard functionality.
