# Date: 2026-03-24

- **What was done:**

Today I started implementing the learning flow after clicking the **Practice** button. The goal was to open a page with structured learning materials instead of using the existing topic page.

At first, I connected the button to the `/topics/:id` route, but quickly noticed that this route is already used by the dashboard. Changing its behavior started breaking the existing functionality, especially navigation from the dashboard.

I also reviewed the Firebase structure and realized that topics and learning content were mixed together. There was no clear distinction between a topic and a lesson, which made it difficult to build a proper learning experience.

- **Decisions / Notes:**

Decided to:

- separate dashboard and learning flows completely
- keep `/topics/:id` unchanged for the dashboard
- introduce a new route structure for learning (`/learn/:topicId`)
- split data into:
  - topics (metadata)
  - lessons (actual learning content)

This gives a much cleaner and scalable structure.

- **Problems / Questions:**

The main issue was that reusing existing routes caused unexpected side effects. It became clear that even small changes could break already working parts of the app.

Also, the Firebase structure needed to be reconsidered, because it did not support multiple lessons inside one topic properly.

- **Plans / Next steps:**

- create a dedicated Lessons page for each topic
- fetch lessons from Firebase subcollections
- implement lesson navigation
- start building completion logic
