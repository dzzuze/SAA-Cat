Today I reviewed a teammate’s pull request [PR #32: Feat/btn-component](https://github.com/dzzuze/SAA-Cat/pull/32) **Feat/btn-component (#32)** where a reusable button component was implemented.

While reading the code I focused on how the component is named and how it behaves when used inside forms. I left two comments suggesting small improvements.

First, I suggested renaming the component using **PascalCase** (`ButtonLoading`) instead of `btn-loading`. In React it is common practice to name components using PascalCase so they are easily distinguishable from regular HTML elements.

Second, I suggested explicitly adding `type={type}` with a default value `type="button"` to the button component. By default, a `<button>` inside a `<form>` behaves as `type="submit"`, which can trigger unintended form submissions when the button is used for other actions.
