# Date: 2026-02-28

- **What was done:**

Today was less about writing new features and more about thinking structurally.

Yesterday we had a call where we discussed routing and agreed that using wrapper components like `ProtectedRoute` and `PublicOnlyRoute` is not the most scalable solution. We decided to move toward a layout-based architecture using React Router’s `<Outlet />`. At the time it sounded simple, but today I realized it’s actually a small architectural shift.

Before changing anything, I mentally mapped how the routing tree should look if we treat layouts as structural layers instead of just guards. I wanted it to feel clean and predictable, not like logic scattered across different wrappers.

Then I created:
- `AuthGateLayout` — to handle `loading` state in one place  
- `ProtectedLayout` — to handle redirecting unauthenticated users  
- `GuestOnlyLayout` — to prevent logged-in users from accessing login/register.
