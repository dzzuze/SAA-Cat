## Discussion

This meeting happened after Anna started implementing the learning flow and noticed that the existing `/topics/:id` route was already tied to the dashboard. Reusing it for learning pages started to break already working behavior, which made it clear that the route structure needed to be reconsidered.

We discussed the difference between:
- a topic as dashboard metadata
- a lesson as actual learning content

At the time those ideas were still partially mixed in Firebase, which made the product logic blurry. After going through examples and edge cases, we agreed that dashboard browsing and guided learning flow should be treated as different parts of the app.

That led to a cleaner plan: keep dashboard routing intact and create a separate learning route structure. We also discussed how lessons should be fetched and how progression might later connect to user progress.

This meeting was valuable because it prevented us from patching over a structural problem with quick fixes.

## Decisions
- Do **not** reuse `/topics/:id` for the guided learning flow
- Keep dashboard logic separate
- Introduce a separate learning route pattern such as `/learn/:topicId`
- Distinguish between **topics** and **lessons** in the data model
